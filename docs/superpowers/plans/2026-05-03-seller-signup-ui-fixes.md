# Seller Signup UI Fixes & Optional Compliance Fields — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix Step 4 cramped layout on seller signup wizard and make GST / FSSAI / bank account / IFSC fields optional across frontend, backend schema, ORM model, and database column.

**Architecture:** Two parallel-safe surface changes. Backend: relax schema (Pydantic) + model (SQLModel) + add Alembic migration to drop NOT NULL on `sellerprofile.bank_account_number` and `bank_ifsc`. Frontend: restructure Step 4 JSX to put Business name and Services Offered on separate full-width rows (drop the `.formGrid` 2-col wrapper), and loosen Step 5 validation so blank compliance fields advance the wizard while filled fields still get format-checked.

**Tech Stack:** FastAPI + SQLModel + Alembic + asyncpg (backend); Next.js 16 App Router + React 19 + TypeScript + CSS Modules (frontend); Pytest + httpx ASGITransport (tests).

**Spec:** `docs/superpowers/specs/2026-05-03-seller-signup-ui-fixes-design.md`
